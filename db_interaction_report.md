# Отчет о взаимодействии модулей приложения с базой данных

## Общая информация
После изменения структуры базы данных все модули приложения были адаптированы для работы с новой схемой. Взаимодействие с БД осуществляется через SQLAlchemy ORM с использованием асинхронных сессий.

## Структура базы данных
Основные таблицы:
- `users` - основная таблица пользователей
- `user_profiles` - профили пользователей с данными для расчетов
- `natal_charts` - натальные астрологические карты
- `psyho_matrices` - нумерологические психоматрицы
- `biorhythms` - расчеты биоритмов
- `magic_profiles` - интегрированные психологические и эзотерические профили
- `optimal_activities` - рекомендации оптимальных активностей
- `recommendations` - итоговые персонализированные рекомендации
- `psychological_tests` - результаты психологических тестов

## Модули и их взаимодействие с БД

### 1. Модуль пользователей (user_services.py)
**Таблица:** `users`, `user_profiles`

**Операции:**
- **Запись/Перезапись:** `create_or_update_user()` - создание или обновление пользователя и его профиля
- **Чтение:** `get_user_profile()` - получение профиля пользователя
- **Перезапись:** `update_user_profession()` - обновление профессиональных данных пользователя

### 2. Модуль натальных карт (chart_services.py)
**Таблица:** `natal_charts`

**Операции:**
- **Запись/Перезапись:** `create_and_save_natal_chart()` - создание и сохранение натальной карты
- **Чтение:** `get_user_natal_chart()` - получение натальной карты пользователя

### 3. Модуль психоматриц (matrix_services.py)
**Таблица:** `psyho_matrices`

**Операции:**
- **Запись/Перезапись:** `calculate_and_save_psyho_matrix()` - расчет и сохранение психоматрицы
- **Чтение:** `get_user_matrix()` - получение психоматрицы пользователя

### 4. Модуль биоритмов (biorhythm_services.py)
**Таблица:** `biorhythms`

**Операции:**
- **Запись/Перезапись:** `calculate_and_save_biorhythms()` - расчет и сохранение биоритмов
- **Чтение:** `get_user_biorhythms()` - получение биоритмов пользователя

### 5. Модуль магических профилей (magic_profile.py)
**Таблица:** `magic_profiles`

**Операции:**
- **Запись/Перезапись:** `MagicProfileService._save_magic_profile_to_db()` - сохранение magic profile в БД
- **Чтение:** `MagicProfileService._get_magic_profile_from_db()` - получение magic profile из БД
- **Удаление:** `MagicProfileService._delete_magic_profile_from_db()` - удаление magic profile из БД

### 6. Модуль рекомендаций (prediction_services.py)
**Таблица:** `recommendations`

**Операции:**
- **Запись/Перезапись:** `generate_and_save_prediction()` - генерация и сохранение предсказания
- **Чтение:** `get_user_predictions()` - получение предсказаний пользователя
- **Чтение:** `get_todays_prediction()` - получение предсказания на сегодня

### 7. Модуль оптимальных активностей (activity_optimizer.py)
**Таблица:** `optimal_activities`

**Операции:**
- **Запись/Перезапись:** `ActivityOptimizer._save_ml_activities_to_db()` - сохранение ML-активностей в БД
- **Чтение:** `ActivityOptimizerService._get_cached_ml_activities()` - получение кэшированных ML-активностей из БД

## Общие принципы взаимодействия с БД

1. **Асинхронность:** Все операции с БД выполняются асинхронно с использованием `async_session`
2. **Контекстные менеджеры:** Для каждой операции создается отдельная сессия БД
3. **Обработка ошибок:** Все операции обернуты в try-except блоки с логированием
4. **Транзакции:** Изменения данных выполняются в рамках транзакций с автоматическим коммитом или откатом
5. **Кэширование:** Для оптимизации производительности используется кэширование результатов в БД

## Заключение
Все модули приложения успешно адаптированы для работы с новой структурой базы данных. Каждый модуль отвечает за определенный тип данных и реализует стандартный набор операций: создание, чтение, обновление и в некоторых случаях удаление записей.
